# Redactar Bug Profesional

## Objetivo

Redactar un bug de forma clara, profesional y entendible para desarrollo, negocio y QA.

## Cuándo usarlo

Úsalo cuando hayas encontrado una incidencia y necesites reportarla en Jira, Azure DevOps, Excel o por correo.

## Prompt
```
Actúa como un QA Engineer con experiencia en reporte de defectos.

Redacta el siguiente bug de forma profesional, clara y estructurada.

Información del defecto:


[PEGA AQUÍ LA INFORMACIÓN DEL BUG]


Considera:

- Título claro.
- Ambiente.
- Módulo o flujo afectado.
- Precondiciones.
- Pasos para reproducir.
- Resultado esperado.
- Resultado actual.
- Impacto en negocio.
- Severidad sugerida.
- Prioridad sugerida.
- Evidencia requerida.
```
