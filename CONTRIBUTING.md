# Guía de contribución

Este repositorio contiene prompts para actividades de QA.

## Reglas básicas

1. Usar nombres de archivos claros y en minúsculas.
2. Separar palabras con guiones.
3. Mantener los prompts simples y fáciles de usar.
4. Incluir siempre variables entre corchetes `[ ]`.
5. Indicar el formato de salida esperado.
6. Revisar que el prompt no contenga información sensible del proyecto.

## Estructura recomendada para cada prompt

```md
# Nombre del Prompt

## Objetivo

## Cuándo usarlo

## Prompt

## Formato de salida esperado

## Variables a reemplazar

## Recomendaciones
```

## Ejemplo de nombre de archivo

```text
generar-casos-prueba-desde-requerimiento.md
```
