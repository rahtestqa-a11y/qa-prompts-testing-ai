# 🧪 QA Prompts Testing AI

Biblioteca inicial de prompts para apoyar actividades de QA usando herramientas de IA como ChatGPT, Claude, Gemini u otras.

Este repositorio es un piloto personal para organizar prompts reutilizables que ayuden a acelerar el análisis, diseño, documentación, automatización y validación de pruebas de software.

## 🎯 Objetivo

Centralizar prompts simples, claros y reutilizables para que cualquier QA pueda usarlos en su trabajo diario, reemplazando la información entre corchetes `[ ]` con datos reales del proyecto.

## 📂 Contenido

| Categoría | Descripción | Uso principal |
|---|---|---|
| Casos de prueba | Prompts para generar casos, escenarios y matrices | Diseño de pruebas |
| Bugs y reportes | Prompts para redactar bugs, clasificar severidad e impacto | Gestión de defectos |
| Análisis y estrategia | Prompts para revisar requisitos, detectar gaps y proponer estrategia QA | Análisis QA |
| Automatización | Prompts para apoyar la creación de scripts y frameworks | QA Automation |
| APIs y servicios | Prompts para diseñar pruebas de APIs REST | API Testing |
| Performance | Prompts para diseñar y analizar pruebas de rendimiento | Performance Testing |
| Regresión y cobertura | Prompts para definir suites y evaluar cobertura | Control de regresión |
| Documentación | Prompts para informes, planes y correos QA | Gestión documental QA |
| Mobile Testing | Prompts para pruebas móviles | Mobile QA |
| AI Testing | Prompts para validar chatbots y soluciones con IA | QA + IA |

## 🚀 ¿Cómo usar estos prompts?

1. Ingresa a la categoría que necesitas.
2. Abre el archivo del prompt.
3. Copia el contenido de la sección **Prompt**.
4. Reemplaza los textos entre `[CORCHETES]` con la información real.
5. Pega el prompt en tu herramienta de IA.
6. Revisa y ajusta la respuesta antes de usarla en el proyecto.

## 🧩 Ejemplo rápido

```text
[PEGA AQUÍ EL REQUERIMIENTO]

Ejemplo:
El sistema debe permitir registrar una solicitud de atención, validar campos obligatorios y generar una confirmación.
```

Con el prompt adecuado, la IA puede ayudarte a generar:

- Casos de prueba.
- Escenarios positivos y negativos.
- Datos de prueba.
- Validaciones QA.
- Riesgos o ambigüedades.
- Reportes o correos profesionales.

## 🧭 Recomendación de uso

Los resultados generados por IA deben ser revisados por un QA antes de ser usados oficialmente. La IA ayuda a acelerar el trabajo, pero no reemplaza el criterio profesional del equipo de calidad.

## 🗺️ Roadmap del piloto

### Fase 1 — Biblioteca inicial
- Crear estructura base del repositorio.
- Documentar prompts principales para QA funcional.
- Agregar prompts para bugs, casos de prueba y documentación.

### Fase 2 — Validación con casos reales
- Probar los prompts con requerimientos reales.
- Medir utilidad, claridad y tiempo ahorrado.
- Ajustar prompts según feedback del equipo QA.

### Fase 3 — Estandarización
- Definir plantilla oficial de prompts.
- Clasificar prompts por tipo de actividad QA.
- Agregar ejemplos de entrada y salida.

### Fase 4 — Escalamiento
- Evaluar uso por otros miembros del equipo.
- Incorporar prompts para automatización, API, performance y AI Testing.
