# Nombre del Prompt

## Objetivo

Describe qué problema ayuda a resolver este prompt.

## Cuándo usarlo

Usa este prompt cuando necesites:

- [USO 1]
- [USO 2]
- [USO 3]

## Prompt
```
Actúa como un QA Engineer con experiencia en pruebas de software.

Tu objetivo es analizar la siguiente información y generar una respuesta clara, ordenada y útil para QA.

Información de entrada:


[PEGA AQUÍ LA INFORMACIÓN]
```

Considera lo siguiente:

- Identifica escenarios principales.
- Identifica escenarios alternos o negativos.
- Indica validaciones importantes.
- Propón datos de prueba si aplica.
- Señala dudas, riesgos o ambigüedades si existen.

## Formato de salida esperado

| Campo | Descripción |
|---|---|
| ID | Identificador del escenario o caso |
| Escenario | Nombre del escenario |
| Descripción | Explicación breve |
| Resultado esperado | Comportamiento esperado |
| Observaciones | Riesgos, dudas o comentarios |

## Variables a reemplazar

- `[PEGA AQUÍ LA INFORMACIÓN]`: información real del proyecto.
- `[USO 1]`: primer uso del prompt.
- `[USO 2]`: segundo uso del prompt.
- `[USO 3]`: tercer uso del prompt.

## Recomendaciones

Revisa siempre la respuesta generada antes de usarla como evidencia formal del proyecto.
